# 03-design

Purpose: store design evidence, design decisions, drawing registers, design reviews and technical coordination material.

Use this predetermined design-stage structure:

- `01-due-diligence/` — existing information, surveys, audits, investigations, site verification, condition reports and early constraints.
- `02-scheme/` — concept / scheme design options, sketches, design briefs, feasibility markups and early stakeholder review material.
- `03-detail/` — developed / detailed design, coordination drawings, design reports, specifications and pre-tender technical material.
- `04-ifc/` — issued-for-construction drawings, specifications, schedules, construction-phase design clarifications and controlled issue sets.
- `05-as-built/` — as-built drawings, record drawings, final markups, asset information and design close-out records.

Preferred workflow:

1. File design material by design maturity, not just by discipline.
2. Keep source documents authoritative; Markdown registers and summaries are retrieval aids only.
3. Maintain drawing / document registers in the most relevant stage folder, or at `03-design/` root if the register covers all stages.
4. Flag unclear document status rather than assuming a drawing is current.
5. Do not delete older revisions unless the PM asks; mark them as superseded / reference where appropriate.
