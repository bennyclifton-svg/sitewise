# 05-procurement

Purpose: store procurement strategy, tender preparation, tender administration, submissions, evaluation and recommendation records.

Use package-specific folders where needed, for example:

`05-procurement/Fire Services Contractor/`

Within each package, use this standard lifecycle structure:

- `01-eoi/` — EOI, market sounding, tender list / shortlist notes.
- `02-tender-pack/` — RFT / RFQ, tender drawings, specifications, returnable schedules and issued tender pack.
- `03-rfi-addendum/` — tender RFIs, clarifications, addenda and tender-period correspondence.
- `04-submissions/` — tender submissions, attachments and extracted text / summaries.
- `05-evaluation/` — price evaluation, non-price evaluation, clarification analysis and normalisation notes.
- `06-recommendation/` — final tender recommendation report and approval material.

Preferred workflow:

1. Draft procurement artefacts early as working files.
2. Keep submissions separate from evaluation material.
3. Prepare concise price and non-price evaluation drafts first.
4. PM reviews / iterates the evaluations.
5. Prepare the tender recommendation report only after the evaluation position is settled.
