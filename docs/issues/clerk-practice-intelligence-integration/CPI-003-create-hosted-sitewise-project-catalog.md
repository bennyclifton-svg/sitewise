# Create Hosted SiteWise Project Catalog

## Parent

Clerk Practice Intelligence Integration PRD.

## What to Build

Create the first hosted SiteWise project catalog in Clerk. The completed slice should let an authenticated user see at least one imported SiteWise project as a first-class project record, with stable workspace path metadata and declared overlays imported from project README/frontmatter where available.

This is a thin vertical path through schema, import logic, backend API, and the authenticated home/project list UI.

## Acceptance Criteria

- [x] Clerk has first-class project records suitable for org-scoped SiteWise projects.
- [x] Project records can store or expose SiteWise workspace path, title/name, archetype, user role, and state.
- [x] One SiteWise project can be imported or seeded into the project catalog.
- [x] The authenticated Clerk home can list the imported SiteWise project.
- [x] The project list does not depend on parsing a README at request time.
- [x] Tenant or user scoping is enforced at the API boundary.
- [x] Backend tests cover project catalog access and overlay metadata.
