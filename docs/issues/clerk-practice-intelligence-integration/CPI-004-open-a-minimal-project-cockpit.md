# Open A Minimal Project Cockpit

## Parent

Clerk Practice Intelligence Integration PRD.

## What to Build

Add the first Clerk project cockpit route for a SiteWise project. The cockpit should be a thin but complete user path: select a project, open a project dashboard shell, see a centre workspace area, see a chat area, and preview one piece of project evidence.

Use the Practice dashboard as product reference, but implement inside Clerk's frontend conventions.

## Acceptance Criteria

- [x] The user can open a SiteWise project from the authenticated project list.
- [x] The project route renders a cockpit-style layout rather than a generic chat page.
- [x] The cockpit shows project identity and overlay status.
- [x] The cockpit has a centre workspace area for dashboard, evidence, and future drafts.
- [x] The cockpit has a visible chat area or chat entry point for the active project.
- [x] The user can preview at least one text-like project evidence item.
- [x] The frontend follows Clerk conventions: React Router route, `api` client, Tailwind/shadcn styling, no imported Practice test harness.
- [x] Manual verification instructions are documented for opening the project cockpit.
