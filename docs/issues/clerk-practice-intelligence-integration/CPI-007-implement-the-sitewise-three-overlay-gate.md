# Implement The SiteWise Three-Overlay Gate

## Parent

Clerk Practice Intelligence Integration PRD.

## What to Build

Implement the SiteWise three-overlay gate in Clerk. Phase-gate workflows must stop when `archetype`, `user_role`, or `state` is missing, blank, unsupported, or TBC. General chat should remain available for non-deliverable questions.

The slice should include backend gate logic and a cockpit-visible way to resolve or display missing overlays.

## Acceptance Criteria

- [x] Clerk can evaluate whether a project has valid `archetype`, `user_role`, and `state` values.
- [x] Phase-gate workflow preflight fails when any required overlay is missing or invalid.
- [x] Failure messages identify exactly which overlay values are missing or invalid.
- [x] General project chat remains available when overlays are incomplete.
- [x] The project cockpit displays overlay status.
- [x] The user can see what must be supplied before running phase-gate workflows.
- [x] Backend tests cover passing, missing, blank, TBC, and unsupported overlay cases.
