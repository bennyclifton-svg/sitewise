# Persist Draft Artefacts With Provenance

## Parent

Clerk Practice Intelligence Integration PRD.

## What to Build

Add first-class draft artefact persistence for SiteWise workflow outputs. A generated workflow result should be saved as a versioned draft artefact with status, workspace path, author, timestamps, model/runtime metadata, seed consulted, evidence refs, and context references.

This slice should deliver the persistence and read path before Create PMP writes real content through it.

## Acceptance Criteria

- [x] Clerk can create a versioned draft artefact record for a project.
- [x] Draft artefacts carry status, version, title/path, author, and timestamps.
- [x] Draft artefacts can store or reference markdown content.
- [x] Draft artefacts can store provenance metadata including model, seed consulted, evidence refs, and context pack refs.
- [x] Draft versioning does not overwrite previous drafts.
- [x] The backend can fetch a project's latest draft artefact for a workflow type.
- [x] Draft artefact access is scoped to the owning user/org/project.
- [x] Tests cover version increments, provenance fields, and access scoping.
