## What to Build

Document and verify the deployment path for the integrated Clerk product on a VPS. The deployment model should serve one Clerk frontend and one Clerk FastAPI backend, keep model and Supabase service credentials backend-only, and use same-origin proxying or equivalent routing so the browser talks to Clerk without exposing privileged secrets.
