# Add Document Register Table And Preview API

## Parent

Clerk Practice Intelligence Integration PRD.

## What to Build

Replace the current evidence-snippet cards in the far-right panel with Practice Intelligence's **document register table**, backed by a documents API that returns register-oriented metadata from ingest (`CPI-017`).

The repository panel remains the upload drop zone (`CPI-014`). After upload and ingest, users see a sortable compact table — not filename cards grouped by document class alone.

## Practice Reference (UI contract)

| Practice component | Register columns (compact repo pane) |
|--------------------|--------------------------------------|
| `DocumentRegisterSurface` in `App.tsx` | **Doc No**, **Title**, **Rev**, **Category** |
| Full (non-compact) mode adds | **Type** (file extension) |

Row data comes from `DocumentRegisterRow` via `parseDocumentMetadata` → `buildDocumentRegisterRow`:

- `documentNumber` → Doc No
- `title` → Title (falls back to filename)
- `revision` → Rev
- `category` → Category (Practice `metadata.discipline`: Architectural, Electrical, Authorities, etc.)
- `type` → Type (PDF, DOCX, …)

Default sort: document number (numeric-aware), then title — same as Practice `compareDocumentRegisterRows`.

## Acceptance Criteria

- [ ] `GET /projects/{project_id}/documents` returns register rows with: `id`, `path`, `name`, `document_number`, `title`, `revision`, `category`, `type`, `size_bytes`, `ingest_status`, and `workspace_path` folder segment.
- [ ] Register fields are populated from `source_documents.document_metadata` produced by `CPI-017`, not re-parsed in the frontend.
- [ ] `GET /projects/{project_id}/documents/{document_id}/preview` returns preview content or a signed download URL for the stored binary.
- [ ] `DocumentRepositoryPanel` renders a **compact register table** with columns **Doc No | Title | Rev | Category** matching Practice repo-pane layout.
- [ ] Table rows are sortable by column (at minimum Doc No and Title).
- [ ] Row click selects the document and opens centre preview in `EvidenceDetailPanel`.
- [ ] Rows with missing metadata show em dash or filename fallback — same graceful degradation as Practice (`documentNumber || "—"`).
- [ ] Inbox items (`_inbox/` path prefix) appear in the table with whatever metadata ingest extracted; category may come from inbox folder hints before Sort Files.
- [ ] Upload drop zone behaviour from `CPI-014` remains on the same panel (table + drop overlay).
- [ ] `EvidenceDetailPanel` shows full preview where supported, plus register metadata header (doc no, title, rev, category, path).
- [ ] Backend tests cover documents list shape, metadata field mapping, and access control.
