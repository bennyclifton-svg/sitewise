# Port Practice Document Metadata Extraction

## Parent

Clerk Practice Intelligence Integration PRD.

## What to Build

Port the Practice Intelligence document identity and register metadata logic into Clerk's Python backend so uploaded and ingested project files produce structured register fields: **document number**, **title**, **revision**, and **category/discipline**.

This is the extraction engine behind the far-right document register table. Practice invested heavily here; Clerk must not reimplement a simplified subset.

## Practice Reference (contract source)

| Practice module | Role |
|-----------------|------|
| `apps/clerk-workspace-dashboard/src/shared/documentMetadata.ts` | Main parser: filename patterns, title-block text, preview snippets, discipline/category inference, canonical filename |
| `apps/clerk-workspace-dashboard/src/shared/documentMetadata.test.ts` | Fixture corpus (CC-A drawings, E## sheets, hydraulic layouts, inbox folder hints, etc.) |
| `apps/clerk-workspace-dashboard/src/shared/fixtures/intake-filename-corpus.json` | Additional filename regression cases |
| `apps/clerk-workspace-dashboard/src/client/appPersistence.ts` | `buildDocumentRegisterRow` maps `DocumentMetadata` → register row |
| `apps/clerk-workspace-dashboard/src/server/documentParsers.ts` | First-page / bounded preview extraction feeding `previewSnippet` |

Clerk already has partial equivalents that must be extended, not left parallel:

- `backend/ingest/drawing_parse.py` — basic filename patterns only
- `backend/ingest/extractors/pdf_drawing.py` — title-block region text extraction without field parsing
- `source_documents.document_metadata` JSONB — currently stores limited drawing fields

## Acceptance Criteria

- [x] New Python module (for example `backend/app/documents/metadata.py` or `backend/ingest/document_metadata.py`) exposes a `parse_document_metadata(...)` function with inputs equivalent to Practice: `file_name`, `filed_path`, optional `preview_snippet`, optional `source_path`.
- [x] Output includes at minimum: `document_number`, `title`, `revision`, `discipline` (category), `canonical_file_name`, `confidence` (`high` / `medium` / `low`).
- [x] **Filename parsing** covers the Practice pattern set: drawing numbers (CC-A-*, E##, H-*, bracket/paren revisions, opaque codes, Rev suffix stems, etc.).
- [x] **Title-block / preview parsing** merges first-page PDF text, DOCX/XLSX bounded preview, and markdown frontmatter/table patterns using the same precedence rules as Practice (`parseFromTitleBlockText`, `parseFromPreview`, `mergeParsedFields`).
- [x] **Category/discipline inference** ports Practice path logic: design/consultant folder slugs, `_inbox/<PACKAGE>/` folder hints, filename keywords, lifecycle folder categories (`Cost`, `Design`, `Authorities`, `Procurement`, `Construction`, `Reporting`).
- [x] **Canonical filename** follows Practice format `{documentNumber} - {title} Rev {revision}.{ext}` when confidence is not `low`.
- [x] Ingest persists parsed fields on `source_documents.document_metadata` (for example `document_number`, `title`, `revision`, `discipline`, `canonical_file_name`, `metadata_confidence`).
- [x] Port Practice fixture cases into pytest (translate JSON corpus or representative parametrized tests); fast suite green without network.
- [x] Documented mapping from Practice TypeScript tests to Python tests so future classifier changes stay aligned.

**Category:** enhancement
**Summary:** Port Practice `parseDocumentMetadata` and discipline inference to Python for register-ready ingest metadata.

**Current behavior:** Clerk extracts basic drawing filename identity and raw title-block text; it does not populate register-grade document number, title, revision, and category fields the way Practice does.

**Desired behavior:** Every ingested project document gets structured register metadata using the same rules and fixture coverage as Practice Intelligence.

**Key interfaces:** `parse_document_metadata`, ingest persist layer writing `document_metadata`, pytest port of `documentMetadata.test.ts` cases.

**Out of scope:** OCR for scanned sheets, DWG geometry parsing, LLM-based metadata guessing.

## Implementation Notes

- Parser module: `backend/ingest/document_metadata.py` — `parse_document_metadata`, `parse_from_title_block_text`, `DocumentMetadata` dataclass.
- Tests: `backend/tests/ingest/test_document_metadata.py` — test function names mirror Practice `documentMetadata.test.ts` cases 1:1 (e.g. `test_parses_cc_a_architectural_drawing_numbers` ↔ *parses CC-A architectural drawing numbers*).
- Corpus fixture: `backend/tests/fixtures/intake-filename-corpus.json` (ported from Practice); round-trip and confidence-distribution tests included.
- Persist wiring delivered in `CPI-018` (`ingest/persist.py` `_register_metadata`).

## Design Notes

- Treat Practice's TypeScript module as the behavioural contract; port logic, do not re-specify from memory.
- Prefer deterministic regex/heuristic parsing over model calls for register fields.
- When preview and filename disagree, follow Practice merge precedence (title block / preview can override weak filename titles).
