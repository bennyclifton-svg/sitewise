# Dependencies and Preflight

## Parent

TCM Stage 0.5 — Ingestion, Classification & Extraction Wiring
([plan](../../plans/2026-06-13-tcm-stage0.5-ingestion-classification.md), Task 0).

## What to Build

Add the OCR/image dependencies the ingestion pipeline needs, confirm the Tesseract
binary is available for manual verification, and confirm that **no Alembic migration**
is required (every column Stage 0.5 writes already exists in the schema).

This is the preflight slice — it unblocks all the implementation issues but ships no
behaviour of its own beyond dependency wiring.

## Files

- Modify: `backend/pyproject.toml` (+ `pytesseract`, `pillow`) and `backend/uv.lock`
- Verify only: `backend/tender/models.py`, system Tesseract install

## Steps

1. `cd backend && uv add pytesseract pillow` — updates `pyproject.toml` + `uv.lock`.
2. `tesseract --version` — confirm the binary exists. If missing:
   - Windows dev: `winget install UB-Mannheim.TesseractOCR` (or the UB Mannheim installer).
   - Linux/VPS: `apt-get install -y tesseract-ocr`.
   - OCR unit tests mock pytesseract, so a missing binary only blocks the manual
     verification (TCM05-008). Record the outcome but do not block other issues.
3. `cd backend && uv run pytest tests/tender/test_migrations.py -v` — confirm schema is
   current (the `tender_pages` table and the `tender_documents.content_hash / doc_type /
   classification_confidence / ocr_applied / page_count / ingest_status` columns already
   exist). If this fails, STOP and reconcile migrations before proceeding.

## Acceptance Criteria

- [ ] `uv run pytest tests/tender/test_migrations.py -v` passes — confirming no new migration is needed.
- [ ] `uv run ruff check tender` still clean.
