# PDF Text Extraction and OCR-Candidate Detection

## Parent

TCM Stage 0.5 — Ingestion, Classification & Extraction Wiring
([plan](../../plans/2026-06-13-tcm-stage0.5-ingestion-classification.md), Task 1).

## What to Build

A pure, side-effect-free helper module `tender/services/pdf.py` that (a) extracts the
text layer per page from PDF bytes, and (b) decides whether a page is an OCR candidate
based on text density (chars per page area) against `tender_ocr_text_density_threshold`.

No DB, storage, or network — these are the testable units the ingestion handler
(TCM05-004) orchestrates. Verifiable on its own via synthetic PyMuPDF PDFs.

## Files

- Create: `backend/tender/services/pdf.py`
- Test: `backend/tests/tender/test_pdf.py`

## Design

- `extract_pages(pdf_bytes: bytes) -> list[PageExtract]` where
  `PageExtract(page_no, text, text_chars, page_area)` (1-based `page_no`,
  `page_area = width * height` in PDF points). Open with
  `fitz.open(stream=pdf_bytes, filetype="pdf")`; always close in `finally`.
- `is_ocr_candidate(page, *, threshold) -> bool`: `True` when
  `page.text_chars / page.page_area < threshold` (and `True` if `page_area <= 0`).

Full reference implementation is in the plan (Task 1, Step 3) — copy it verbatim.

## Agent Brief

> *This was generated by AI during triage.*

**Category:** enhancement
**State:** ready-for-agent
**Summary:** Add pure PDF text-layer extraction and density-based OCR-candidate detection.

**Current behavior:**
TCM ingestion has no dedicated PDF helper for extracting per-page text or identifying pages that need OCR. The ingestion handler would have to mix parsing logic into orchestration.

**Desired behavior:**
A side-effect-free PDF helper exposes a small public interface for extracting one record per page and checking whether a page falls below the configured text-density threshold. The helper should not import database, storage, or network modules.

**Key interfaces:**
- `PageExtract` - immutable per-page extraction record with 1-based page number, stripped text, text character count, and page area.
- `extract_pages()` - accepts PDF bytes and returns ordered page extraction records.
- `is_ocr_candidate()` - applies text density threshold logic and treats invalid page area as OCR-required.

**Acceptance criteria:**
- [ ] Text-layer PDFs return extracted text and 1-based page numbers.
- [ ] Text-heavy pages are not OCR candidates at the configured threshold.
- [ ] Image-only pages are OCR candidates at the configured threshold.
- [ ] The helper remains pure: no database, storage, or network imports.
- [ ] Focused tests and lint pass.

**Out of scope:**
- Rendering pages to images.
- Running OCR.
- Persisting extracted pages.
