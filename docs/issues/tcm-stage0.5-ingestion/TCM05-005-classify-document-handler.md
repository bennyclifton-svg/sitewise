# Implement the `classify_document` Handler

## Parent

TCM Stage 0.5 — Ingestion, Classification & Extraction Wiring
([plan](../../plans/2026-06-13-tcm-stage0.5-ingestion-classification.md), Task 4).

## What to Build

A new `classify_document` worker stage (PRD §9.1) that adjudicates a document's
`doc_type` over the `DOC_TYPES` enum using the **first two pages' text + filename**,
writes `doc_type` + `classification_confidence` onto the `TenderDocument`, advances
`quote.stage='extract_line_items'`, and enqueues the next job.

Uses the existing LLM client's `adjudicate(question, choices, evidence, context, *,
prompt_version, model_key)` with `model_key="tender_model_adjudicate_small"` (the
small/cheap model — PRD §7.5 cost tiers).

Below the PRD 0.8 confidence threshold the stage still records the result and proceeds;
the existing QA console surfaces low-confidence `document_classification` items
(`schemas.QAReviewItem`). Hard QA-gating is **out of scope** here.

## Design

- Handler: `async def classify_document(session, job, *, llm_client=None) -> None`.
- Load doc by `job.payload["document_id"]`; read first 2 `tender_pages.text_content`
  ordered by `page_no` (`.limit(2)`).
- `evidence = {"filename": document.original_filename, "first_pages_text": [...]}`.
- Build context via the shared loader `tender.services.context.context_for_quote`
  (created in TCM05-006).
- Call `adjudicate("Classify this tender document into one doc_type.", list(DOC_TYPES),
  evidence, context, prompt_version="0.1.0", model_key="tender_model_adjudicate_small")`.
- Write `document.doc_type = decision.choice`,
  `document.classification_confidence = decision.confidence`.
- Advance `quote.stage='extract_line_items'`; `jobs.enqueue(kind='extract_line_items', ...)`.

Prompt file `classify_document_v0.1.0.md` carries a semver header (see plan Task 4,
Step 3) instructing single-label doc_type classification, no arithmetic.

Full reference implementation: plan Task 4, Step 3.

**Summary:** Add a worker stage that classifies tender documents into one `doc_type`.

**Current behavior:**
There is no registered or implemented `classify_document` handler, so the ingestion pipeline cannot adjudicate document type or advance to line-item extraction.

**Desired behavior:**
The handler loads the target document, sends filename plus the first two pages of text to the LLM adjudication interface with the small tender adjudication model, writes the chosen document type and confidence, advances the quote to extraction, and enqueues `extract_line_items`.

**Key interfaces:**
- `classify_document()` - worker handler for the `classify_document` job kind.
- `TenderLLMClient.adjudicate()` - classification boundary, injected in tests.
- `DOC_TYPES` - allowed classification choices.
- `context_for_quote()` - shared quote context loader.

- [ ] Focused tests and lint pass.
