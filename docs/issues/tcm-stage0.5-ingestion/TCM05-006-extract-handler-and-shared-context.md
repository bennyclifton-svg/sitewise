# `extract_line_items` Worker Handler + Shared Context Loader

## Parent

TCM Stage 0.5 — Ingestion, Classification & Extraction Wiring
([plan](../../plans/2026-06-13-tcm-stage0.5-ingestion-classification.md), Task 5).

## What to Build

`backend/tender/services/extraction.py` is already implemented and tested — it is a
**pure** function `extract_line_items(*, pages, context, llm_client, stated_total_cents, ...)`
returning an `ExtractionResult` (reconciliation + confidence gating, PRD §9.2). It is not
wired to the worker and has no persistence.

Build the `(session, job)` worker handler that loads the document's pages from the DB,
calls the pure function, persists `TenderLineItem` rows and any `TenderFlag`
(arithmetic-inconsistency) rows, then chains to `embed_items`. Also extract the
duplicated `_context_for_quote` query into a shared module (DRY — `mapping.py` and
`silence.py` both have private copies).

## Design

`context.py` (lifted verbatim from `mapping.py:653`):

```python
async def context_for_quote(session, quote_id) -> ProjectContext:
    result = await session.execute(
        select(TenderComparison.context)
        .join(TenderQuote, TenderQuote.comparison_id == TenderComparison.id)
        .where(TenderQuote.id == quote_id))
    return ProjectContext.model_validate(result.scalar_one())
```

`extraction_handler.extract_line_items_job`:
- Load doc by `job.payload["document_id"]`; load its `TenderQuote` (for `stated_total_cents`).
- Build `TenderDocumentPage(document_id=str(doc.id), page_no=p.page_no,
  text_content=p.text_content, image_path=p.image_path)` for each `tender_pages` row
  ordered by `page_no`.
- `result = await extract_line_items(pages=..., context=..., llm_client=...,
  stated_total_cents=quote.stated_total_cents)`.
- **Idempotent re-run:** delete prior `TenderLineItem` rows for this `document_id` first.
- Persist each `reconciled.item` as a `TenderLineItem` (map `bbox` via
  `item.bbox.model_dump()`, `extraction_confidence = reconciled.effective_confidence`,
  `section_path or None`).
- Persist each `result.flags` entry as a `TenderFlag` (`flag_type='arithmetic_inconsistency'`,
  `severity='warning'`, evidence carrying page_no/expected/actual/delta_ratio).
- Advance `quote.stage='embed_items'`; `jobs.enqueue(kind='embed_items', ...)`.

Then refactor `mapping.py` and `silence.py` to delegate their private
`_context_for_quote` to `context.context_for_quote`, and run their tests to confirm green.

Full reference implementation: plan Task 5, Step 3.

## Agent Brief

> *This was generated by AI during triage.*

**Category:** enhancement
**State:** ready-for-agent
**Summary:** Wrap the pure line-item extraction service in a worker handler and share quote context loading.

**Current behavior:**
Line-item extraction exists as a pure service but is not callable as a worker stage. Context loading is duplicated privately in mapping and silence services.

**Desired behavior:**
The worker handler loads a document, its quote context, and ordered page records, calls the pure extraction service, replaces prior extracted line items for idempotent reruns, persists line items and arithmetic flags, advances the quote to embedding, and enqueues `embed_items`.

**Key interfaces:**
- `context_for_quote()` - shared loader returning `ProjectContext` for a quote.
- `extract_line_items_job()` - worker handler for the `extract_line_items` job kind.
- `extract_line_items()` - existing pure extraction service, unchanged.
- `TenderLineItem`, `TenderFlag`, and `TenderDocumentPage` persistence shapes.

**Acceptance criteria:**
- [ ] Shared context loader exists and mapping/silence delegate to it.
- [ ] Handler loads ordered pages and calls the pure extraction service.
- [ ] Handler replaces prior line items for the document on rerun.
- [ ] Handler persists line items and arithmetic inconsistency flags.
- [ ] Handler advances the quote to `embed_items` and enqueues `embed_items`.
- [ ] Focused tests and lint pass.

**Out of scope:**
- Changing extraction prompt/schema/reconciliation logic.
- Implementing embedding.
