# Register Handlers and Auto-Chain the Front Half

## Parent

TCM Stage 0.5 — Ingestion, Classification & Extraction Wiring
([plan](../../plans/2026-06-13-tcm-stage0.5-ingestion-classification.md), Task 6).

## What to Build

Register the three new/wrapped handlers in `worker.HANDLERS` and verify end to end that
an uploaded quote auto-chains `ingest_document → classify_document → extract_line_items
→ embed_items` with no manual `retry/{stage}` clicks. Each handler already enqueues the
next stage on success (TCM05-004/005/006); this issue closes the registry gap that
currently makes `classify_document` and `extract_line_items` fail with
`unknown job kind`, and proves the chain.

> Why this matters: today only `run_expectations → infer_silence` chains; per-quote
> stages were triggered by hand. This is the fix for the "I had to click every button"
> behaviour observed during testing.

## Files

- Modify: `backend/tender/worker.py` (imports + `HANDLERS` dict)
- Test: `backend/tests/tender/test_worker.py` (extend) + `backend/tests/tender/test_worker_chain.py` (new)

## Design

In `worker.py` add imports and entries:

```python
from tender.services.classification import classify_document
from tender.services.extraction_handler import extract_line_items_job
from tender.services.ingestion import ingest_document

HANDLERS = {
    "ingest_document": ingest_document,
    "classify_document": classify_document,
    "extract_line_items": extract_line_items_job,   # job kind -> *_job handler
    "embed_items": embed_items,
    "map_items": map_items,
    "infer_silence": infer_silence,
    "assemble_report_draft": assemble_report_draft,
    "generate_flags": generate_flags,
    "run_analysis": run_analysis,
    "run_expectations": run_expectations,
}
```
