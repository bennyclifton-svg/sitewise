# Manual Real-Document Verification

## Parent

TCM Stage 0.5 — Ingestion, Classification & Extraction Wiring
([plan](../../plans/2026-06-13-tcm-stage0.5-ingestion-classification.md), Task 7).

## What to Build

A human-in-the-loop verification pass over the **real** OCR + render + LLM path that the
unit tests mock. This is HITL because it needs the Tesseract binary, a live
`OPENAI_API_KEY`, real PDFs, and human judgement on the rendered output and extracted
line items. No production code changes — the deliverable is recorded verification results.

## Prerequisites

- TCM05-007 merged (handlers registered + chaining).
- Tesseract installed on the host (TCM05-001).
- Valid `OPENAI_API_KEY` in `backend/.env`.

## Steps

1. Terminal A: `cd backend && uv run uvicorn app.main:app --reload`.
2. Terminal B: `cd backend && uv run python -m tender.worker` — expect
   `Tender worker ready ... handlers=[...]` listing all ten kinds including
   `classify_document` and `extract_line_items`.
3. In the cockpit, upload the three pilot PDFs to the Enmore quote and trigger
   `ingest_document` (or re-upload).
4. Watch Terminal B: `tender_job_claimed → tender_job_done` should flow
   `ingest_document → classify_document → extract_line_items → embed_items` **with no
   manual clicks**.
5. DB checks:
   - `tender_documents`: `ingest_status='ingested'`, `page_count` matches the PDF,
     `doc_type` set, `classification_confidence` populated.
   - `tender_pages`: one row per page, `image_path` set; open one PNG from storage and
     confirm it is a 150-DPI render.
   - `tender_line_items`: rows present for the quote with sane `amount_cents`.
6. Negative checks: upload a non-PDF (→ `unsupported_format`); re-upload an identical
   PDF to the same quote (→ `duplicate`).

## Acceptance Criteria

- [ ] Worker startup lists all ten handlers.
- [ ] The three pilot PDFs flow through the full front-half chain with no manual stage clicks.
- [ ] `tender_documents` / `tender_pages` / `tender_line_items` populated and sane (per checks above).
- [ ] One rendered PNG visually confirmed as a 150-DPI page image.
- [ ] Negative paths confirmed: `unsupported_format` and `duplicate`.
- [ ] Results (pass/fail per check) recorded in a "Stage 0.5 verification" section appended to [docs/plans/tcm/01-stage0-consolidation.md](../../plans/tcm/01-stage0-consolidation.md).
