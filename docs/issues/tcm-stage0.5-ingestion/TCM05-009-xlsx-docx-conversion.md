# XLSX/DOCX Conversion (Optional)

## Parent

TCM Stage 0.5 — Ingestion, Classification & Extraction Wiring
([plan](../../plans/2026-06-13-tcm-stage0.5-ingestion-classification.md), Task 8).

## What to Build

Optional: accept XLSX/DOCX quote documents by converting them to a normalised PDF before
the existing ingestion pipeline runs (PRD §7.4). Only needed if the pilot receives
non-PDF quotes. **Deferred / HITL** because it introduces heavy infrastructure
(LibreOffice headless) on both the host and the worker Docker image — confirm with the
user before starting.

Until built, non-PDF inputs keep the existing `ingest_status='unsupported_format'`
behaviour (M1 Decision 5), so this is purely additive.

## Files

- Modify: `backend/tender/services/ingestion.py` (pre-step before the PDF path)
- Modify: `deploy/docker/backend.Dockerfile` (install `libreoffice`)
- Test: `backend/tests/tender/test_ingestion.py` (conversion branch, with `soffice` mocked)

## Design

- In `ingest_document`, if mime is XLSX/DOCX: run
  `soffice --headless --convert-to pdf --outdir <tmp> <input>` (subprocess, injected so
  tests mock it), then feed the produced PDF bytes into the existing Task-3 path.
- Record provenance (original mime/filename) on the document.
- Keep `unsupported_format` for anything still unconvertible.
- LibreOffice must be present on the host/VPS and in the worker image.

## Acceptance Criteria

- [ ] An XLSX and a DOCX quote each convert to PDF and ingest into `tender_pages`.
- [ ] The conversion subprocess is injected/mocked in unit tests (no LibreOffice in CI unit runs).
- [ ] Unconvertible inputs still resolve to `unsupported_format`.
- [ ] `deploy/docker/backend.Dockerfile` installs LibreOffice for the worker image.
- [ ] `uv run pytest tests/tender/test_ingestion.py -v` passes; `uv run ruff check tender` clean.

## Agent Brief

> *This was generated by AI during triage.*

**Category:** enhancement
**State:** ready-for-agent
**Summary:** Convert XLSX/DOCX quotes to PDF via LibreOffice headless before ingestion.

**Current behavior:**
Non-PDF documents are marked `unsupported_format`, including DOCX and XLSX quotes that could be normalised to PDF for the existing page-render pipeline.

**Desired behavior:**
DOCX and XLSX inputs download normally, run through an injected LibreOffice conversion boundary, and then continue through the existing PDF ingestion path. Conversion failures still resolve to `unsupported_format`. The worker image includes LibreOffice for real deployments, while unit tests mock the converter.

**Decision:**
The issue was optional because of the LibreOffice footprint. The maintainer requested all nine Stage 0.5 issues be implemented, so this is promoted to `ready-for-agent`.

**Key interfaces:**
- `ingest_document(..., converter=None)` - injected conversion boundary.
- LibreOffice `soffice --headless --convert-to pdf` - real default converter.
- Backend Dockerfile - runtime image must include LibreOffice.

**Acceptance criteria:**
- [ ] DOCX and XLSX inputs can convert to PDF and ingest into pages.
- [ ] Conversion is mocked in unit tests.
- [ ] Conversion failure marks the document `unsupported_format`.
- [ ] Runtime image installs LibreOffice.

**Out of scope:**
- Spreadsheet-native structured extraction.
- Supporting legacy `.doc`/`.xls` unless explicitly added later.
